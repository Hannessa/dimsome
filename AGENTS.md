## Commenting Style

- When adding or significantly changing code, also add short one-line comments regularly throughout the new logic.
- Prefer comments that explain intent, flow, or why a block exists, especially in more complex sections.
- Add comments before loops, conditionals, state transitions, async flows, and non-obvious data transformations.
- Keep comments concise and practical. Describe what the code is doing in plain language without repeating every trivial statement.
- Use `//` comments in JavaScript and inside Vue `<script>` blocks. In CSS, use `/* */` comments. In HTML, use `<!-- -->` comments.
- Keep the comment spacing consistent so longer functions are broken up into readable chunks.
- When editing existing code, preserve useful comments and extend the same style into any new additions.
